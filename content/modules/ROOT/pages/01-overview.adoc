== The scenario

Last year your organization was exposed by a security breach on the platform.
The Security Task Force managed to trace back the actions taken by the hacker.
They identified that the hacker got access to one of the Tekton pipelines and used it to inject malicious code in one of the critical applications.
Action was taken and security was re-enforced on the pipeline.

Yesterday, the Network & Security team reported another major incident on the production line.
They claim that an application hosted on an OpenShift cluster was again compromised and important data might have been leaked from the databases.

The team checked and is absolutely sure that the new hacker couldn't have tampered with the secured pipeline.
But they haven't been able to find the security hole that was exploited.

== Your mission

Your objective, as a DevSecOps expert, is to perform the same actions the hacker did.

The 'Customers' application is the one that was attacked.
Unfortunately, in the panic, the Infrastructure Security team destroyed the container image and the version of the application modified by the hacker disappeared, making forensics much more difficult.
The Customers database has been migrated so if the application runs again, it wouldn't leak any sensitive information anymore.

Use the available tools to investigate and discover the security weaknesses.

== Learning objectives

By the end of this challenge, you will be able to:

* Identify security vulnerabilities in OpenShift applications
* Analyze CI/CD pipeline security weaknesses
* Use Red Hat Advanced Cluster Security to detect threats
* Understand how attackers exploit container and Kubernetes vulnerabilities
* Apply DevSecOps best practices to secure applications
* Investigate and trace security incidents

== Expected outcomes

After completing this challenge, you will have:

* Discovered multiple security flags by exploiting vulnerabilities
* Gained practical experience with security tools (ACS, GitOps, Tekton)
* Understood common attack vectors in cloud-native environments
* Learned how to identify and prevent similar breaches
* Developed stronger DevSecOps investigation skills

== Challenge flow

This CTF consists of:

1. **Main Quest** -- 6 sequential missions that unlock as you progress (530 points total)
2. **Side Missions** -- 5 parallel missions that can be completed anytime (250 points total)
3. **Point Zero** -- Initial reconnaissance mission (10 points)

Total possible points: 790

Your score depends on:

* Flags discovered (each flag has a point value)
* Speed (fastest completion wins ties)
* Hints used (hints cost points but help you progress)

Estimated time: 2-4 hours (depending on experience level)
