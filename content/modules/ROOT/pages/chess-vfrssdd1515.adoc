=  Chessmaster

== Principle

Thanks to the information leaked as shown in the Grizzly Credentials challenge, the hacker got itself the priviledge to modify the pipeline.

What was modified ?

The git-clone task.

Even though the pipeline triggered by the hacker was sourcing a Git repository via http, it is still configured with an SSH workspace.
The presence of that workspace, which mounts the SSH Secret that contains the private key(s), allowed the hacker to copy those keys in the /target/classes directory of the application, from which it could retrieve those keys and have them send over by the application at startup.

The maven "package" and "build and sign image" phases dit the rest.
At the end, the file could be read from within the application and sent over to a remote server.

Notice that the hacker modified the pipeline but didn't run it againt to avoid raising suspition.

Instead; they waited for you to do so ....
